# Litom (Music Game)

## 🎮 概要
高校時代に「ゲームエンジンに頼らず、低レイヤーのロジックから自分で実装したい」という動機で開発しました。
Processingをベースにしつつ、当時の知識の範囲内でオブジェクト指向的で可読性の高いコードを書きました。

## 🚀 技術的なアピールポイント

### 1. 独自譜面フォーマットの策定とパーサー実装
既存のフォーマットを使用せず、独自のテキスト形式 `.aof` を定義しました。
楽曲のBPM、オフセット、ノーツ配置（通常ノーツ・ロングノーツ）を記述する仕様を策定し、それを読み込んでオブジェクトに変換するパーサー（`Music` クラス）をフルスクラッチで実装しています。

### 2. オブジェクト指向による拡張性の高い設計
* **シーン管理**:
    `Scene` 抽象クラスと `SceneManager` により、タイトル、選曲、プレイ画面、リザルトといった画面遷移を疎結合に管理しています。
* **ノーツの継承構造**:
    基底クラス `Note` を継承して `LongNote` を作成し、判定ロジックや描画処理を共通化しつつ、長押し特有の処理（中間判定など）を差分実装しています。

## 🛠 開発環境
* **言語**: Java (Processing 3)
* **プラットフォーム**: Windows
* **ツール**: Processing IDE

## 📂 ディレクトリ構成
* `MusicGame.pde`: エントリーポイント
* `GameMain.pde`: ゲームのメインループと判定ロジック
* `SceneManager.pde`: 画面遷移の管理
* `data/`: 楽曲、画像、譜面データ等のアセット

> **Note**
> リポジトリに含まれる楽曲ファイルおよび画像ファイルは、著作権への配慮により本リポジトリからは除外しています。
> ビルドおよび実行には別途楽曲ファイルおよび画像ファイルが必要です。
